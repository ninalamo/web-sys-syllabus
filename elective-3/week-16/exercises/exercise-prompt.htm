Week 16: Capstone Final — Exercises

In-Class Exercise (55 min): Capstone Presentations

Deliverables Checklist

  1. Deployed full-stack app — live URL
  2. Source code — clean repo with README
  3. Documentation — architecture diagram, API docs, setup instructions
  4. CI/CD pipeline — automated build, test, deploy
  5. Test suite — unit, integration, E2E + coverage report
  6. 10-minute demo — live walkthrough
  7. 5-minute Q&A — defend architectural decisions

Presentation Format

  • 10-min demo: Show the live app, walk through key features
  • 5-min Q&A: Defend your architecture choices
  • 2-min feedback: Instructor and peer feedback

Oral Defense Questions to Prepare For

  1. Why did you choose this architecture?
  2. Why JWT over cookie-based auth?
  3. What would you do differently?
  4. How does data flow from frontend to database?
  5. What AI-generated code did you use, and can you explain it?
  6. What tradeoffs did you make?
  7. How is your app secured?
  8. What tests do you have, and what do they cover?

Demo Disaster Prep

  • Record a video of your demo beforehand
  • Take screenshots of key features
  • Have a local fallback ready
  • Test the live deployment right before your presentation
